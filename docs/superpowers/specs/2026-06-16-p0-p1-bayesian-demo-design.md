# P0/P1 Bayesian Rescue Demo Design

## Goal

Upgrade the emergency commander into a defensible, visual experiment: a full discrete Bayesian network with expert and learned CPT modes, coordinate-aware A*, independent drone routing, stateful dynamic dispatch, credible evaluation, and a Streamlit command-center demo.

## Bayesian Network

The network has explicit states, edges, and CPTs:

- `hazard_intensity -> building_damage, fire_severity, road_damage`
- `fire_severity -> smoke_level, trapped_people, road_passable`
- `building_damage, sos_signal, human_activity, smoke_level -> trapped_people`
- `road_damage, congestion, drone_road_report -> road_passable`

All evidence nodes use `low / medium / high`; `drone_road_report` uses `blocked / uncertain / open`; query nodes use `no / yes`. Fixed mode loads expert CPTs. Learned mode estimates every CPT with source weights and Dirichlet smoothing. Exact enumeration handles missing evidence and returns posterior probabilities plus evidence contribution traces.

## Data And Evaluation

The experiment downloads a bounded USGS earthquake catalog CSV and maps magnitude/depth into `hazard_intensity`. xBD and SpaceNet definitions anchor the state vocabulary and provenance, but no imagery is downloaded. The remaining joint variables are generated by ancestral sampling from documented expert CPTs with configurable label noise, missingness, and class imbalance. Every row records `public_anchor`, `expert_cpt`, and `simulated` provenance.

Five-fold cross-validation compares expert and learned CPTs using Brier score, accuracy, F1, ROC-AUC, calibration bins, and missing-evidence robustness. Public-anchor metrics are reported separately from simulated-label metrics. Experiment artifacts include configuration, downloaded data checksum, learned CPTs, fold metrics, aggregate metrics, calibration data, and a Markdown report.

## Routing And Dispatch

The scenario adds a `nodes` coordinate map and separate `roads` and `air_routes`. Ground A* uses Euclidean travel-time lower bounds; drone A* uses only air corridors. Results expose expanded-node counts and heuristic usage.

Each unit tracks `status`, `current_node`, `position`, `capacity`, `onboard`, `current_task`, and remaining travel/service time. Events advance the simulation clock before applying evidence or map changes. Units can continue, replan, rescue, return to hospital when carrying survivors, and become available again. Active assignments are preserved unless invalidated.

## Visual Demo

The Streamlit UI uses an industrial emergency-operations aesthetic: warm paper background, charcoal panels, hazard orange, signal cyan, and compact monospace telemetry. It contains:

- Interactive disaster map with ground roads, air routes, units, hazards, and selected paths.
- Fixed/learned mode switch and event timeline playback.
- Posterior probability, evidence contribution, assignment, ETA, risk, and unit-state panels.
- Experiment page with fold metrics, calibration chart, confusion matrix, CPT provenance, and resource usage.

The demo is validated through automated tests, a headless Streamlit health check, and browser inspection of the loaded page and event interactions.

## Acceptance Criteria

- Exact posterior changes correctly under evidence and missing evidence.
- Learned mode loads learned CPTs, not linear weights, and improves aggregate held-out Brier or reports the measured result honestly.
- Ground A* uses a nonzero admissible heuristic and drones never traverse ground roads.
- Road collapse reroutes affected vehicles from their current state; mission completion updates unit state and availability.
- The same scenario runs in fixed and learned modes with a stable JSON output contract.
- Streamlit renders the map, metrics, timeline, and experiment comparison on the M3/16GB machine.
